一個目的是你可以回到之前的 tag 地方，去看某個版本的 code 長怎麼樣？
因為你一旦繼續下去之後，你以前怎麼寫了到底怎樣，你其實看不到。
所以 git tag 讓你可以切到某個地方。

可是你如果從這邊，你要從這個版本的 code 要繼續寫，然後試著去 commit 進去，它就會跟你說不，你不能改變意思。

所以你如果要這樣做的話，如果你要從這個 tag 開始寫的話，你就要開一個新的 branch，到時候用 branch 跟 branch 之間去 merge 或 rebase 來解決這件事情。
那這是有關於 tag 的一些補充。
