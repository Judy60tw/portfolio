那個……有同學在問幾件事情。
第一件事情，就是剛剛講到「期末」的部分。我有提到，我們會要求大家直接把程式 deploy 到網路上去試用之外，我們也會有一位助教幫大家把 code 下載下來，在我們的 localhost 安裝，然後去跑。

當然，也許大家還不知道我接下來要講什麼，不過——你們的服務如果會牽涉到一些第三方服務（比方說你用了 GPT 的 API），大家應該知道，要用 GPT 不是隨便就能用的，你必須要有個 key 去打他的 API。

所以同學就會擔心：「那你們的 key 要不要給我們？」
當然 不用 這樣子啦！
我們之後會跟大家講，在一個比較完整的服務裡面，會有所谓的 .env 這種環境設定檔。在這個環境設定檔裡面，會去設定你所有的 key 啦、這些秘密的東西。
請你們在上傳的時候，千萬不要 把這個東西上傳上去！因為不只我看到，你基本上讓所有看到你 report 的人都知道了。

所以我們到時候會繼續告訴大家，簡單講就是：
我們會用我們自己的 key 去打你們的服務。所以你們不要在 code 裡面把 key 硬寫死，這也是很蠢的寫法，我看過太多人這樣做了……
總之，key 應該是要從環境變數讀進去的。我相信到那個階段，大家應該不會犯這種低級錯誤啦～不過總之不用擔心。

如果真的有問題，我們還是可以跟你私下約，想辦法讓你協助我們在我們的機器把你的服務跑起來。
我覺得能夠在 local 的機器重新現（重現）你的服務、重新安裝你的開發環境，是非常重要的一件事。
因為你不希望你開發了一個東西，結果以後團隊有新人進來，他要搞一個月都裝不起來。

所以讓一個新的成員可以很快重現這件事情，是很重要的。
因此我們一定會在 localhost 去批改。
